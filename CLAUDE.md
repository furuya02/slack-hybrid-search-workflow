# CLAUDE.md

## 全ての作業に共通した指示

* 指示された作業が多岐にわたる場合は、tmp/todo.mdにリストを追記し、チェックを付けながら逐次すすめる
* レスポンスは、常に日本語
* 不明な点があった場合、必ず「質問」してから作業を開始
* 作業開始前に、作業内容を整理・提示し、「これでいですか？」確認をする
* 作業指示が、「〜〜できますか？」のような質問形式になっている場合は、実作業は実施せず、質問に答えるのみとする
* PR作成やCommitは、明確な指示がない限り実施しない
* 【ultrathink】重要な判断が必要な場合:
    - 速度よりも正確性を優先
    - 最低2つ以上の選択肢を検討
    - 選択理由を明示してから実行
    - トレードオフを理解した上で決定
    
## 作業記録
* 作業を実行するたびに、与えられた「指示」とその「レスポンス・作業内容」を tmp/instructions_history.mdに追記する
* 指示に画像が添付された場合は、その画像も保存し、instructions_history.mdにリンクを入れる
* 日時は、現在の日時を正確に取得する
* 「指示」には、日時(YYYY/MM/DD hh:mm)とともに追記する
* 「レスポンス・作業内容」は、やり取りした全ての内容を追記してください
* 「レスポンス・作業内容」は、可能な限り詳細に記録してください
* 画像を保存する場合は、tmp/instructions_history_YYYY.MM.DD_hh:mm.ss.pngという名前にする
* ローテーション tmp/instructions_history.mdは、500KByteを超えた時点で、tmp/instructions_history_YYYY.MM.DD_hh:mm.mdにコピーして、0バイトにクリアする

## AWSコマンドの実行 

* AWS コマンドを実行する際は環境変数を読み込む

```
source .env && aws s3 ls
set -a && source .env && set +a && aws s3 ls
```
* リージョンが指定されていない場合は、ap-northeast-1とする
